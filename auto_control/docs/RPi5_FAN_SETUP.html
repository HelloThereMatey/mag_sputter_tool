Enable Fan Control in Raspberry Pi OS

The Raspberry Pi OS has built-in fan control in raspi-config, but it requires the proper device tree overlay to be enabled.

Step 1: Edit the Device Tree Configuration

Open the boot configuration file:

sudo nano /boot/firmware/config.txt

Add these lines at the end of the file:

# Enable PWM fan on GPIO 18 (RPi5 fan header) dtoverlay=pwmfan,gpiopin=18,temp=55000
Explanation:
  • dtoverlay=pwmfan → Enables the PWM fan device tree overlay
  • gpiopin=18 → Uses GPIO 18 (the RPi5 fan header PWM pin)
  • temp=55000 → Fan starts at 55°C (in millidegrees Celsius)

Save and exit: Press Ctrl+X, then Y, then Enter

Step 2: Reboot

Reboot the Raspberry Pi for the changes to take effect:

sudo reboot

Wait for the RPi to restart and reconnect via SSH.

Step 3: Verify Fan Control is Enabled

Check if the fan sysfs interface appeared:

ls -la /sys/class/pwm/pwmchip*/ ls -la /sys/class/hwmon/hwmon*/

Look for pwmfan entries:

find /sys -name "*pwmfan*" 2>/dev/null
✅ If you see pwmfan entries, the device tree overlay loaded successfully!

Step 4: Check raspi-config

After reboot, the fan option should appear in raspi-config:

sudo raspi-config

Navigate to:

  1. Performance Options
  2. Fan

You should now see options to:

  • Enable/disable automatic fan control
  • Set temperature thresholds
💡 Once enabled, the OS will automatically control the fan based on CPU temperature!

Alternative: Manual PWM Control via sysfs

If raspi-config still doesn't show the option, you can control the fan manually:

# Find the pwm device ls /sys/class/pwm/ # Export and control it directly echo 0 > /sys/class/pwm/pwmchip0/export echo 1000000 > /sys/class/pwm/pwmchip0/pwm0/period echo 1000000 > /sys/class/pwm/pwmchip0/pwm0/duty_cycle # 100% speed echo 1 > /sys/class/pwm/pwmchip0/pwm0/enable
What this does:
  • Exports PWM control to userspace
  • Sets PWM period to 1,000,000 nanoseconds (1 kHz)
  • Sets duty cycle to 1,000,000 (100% = full speed)
  • Enables the PWM output

Troubleshooting

If fan control still doesn't work:

Check Device Tree is Loaded

dtc -I fs /sys/firmware/devicetree/base | grep -i fan

Check for Errors in Boot Log

dmesg | grep -i pwm dmesg | grep -i fan

Try Alternative GPIO Pin

Some RPi5 boards use GPIO 17 instead of GPIO 18:

  1. Edit /boot/firmware/config.txt
  2. Change gpiopin=18 to gpiopin=17
  3. Reboot
⚠️ If neither GPIO 17 nor 18 works, verify your fan header is correctly connected to the RPi5 "FAN" port!

Permanent Fan Speed (If raspi-config Not Available)

Create a systemd service to run the fan control script at boot:

Create systemd Service File

sudo nano /etc/systemd/system/fan-control.service

Paste the following content:

[Unit] Description=RPi5 Fan Control After=network.target [Service] Type=oneshot ExecStart=/usr/bin/python3 /home/sput/Documents/sputter_control/auto_control/python/fanRun.py Restart=always RestartSec=5 User=root [Install] WantedBy=multi-user.target

Save and exit: Press Ctrl+X, then Y, then Enter

Enable and Start the Service

sudo systemctl daemon-reload sudo systemctl enable fan-control.service sudo systemctl start fan-control.service

Check the service status:

sudo systemctl status fan-control.service
✅ The fan will now start automatically on every boot!

Quick Reference

Fan Header Pin Layout:
  • Pin 1: GND (black wire)
  • Pin 2: 5V (red wire)
  • Pin 3: PWM signal (yellow/white wire) → GPIO 18
  • Pin 4: Tachometer/feedback (blue wire, optional)
Useful Commands:
# Check CPU temperature vcgencmd measure_temp # Check if fan is spinning (tachometer feedback) cat /sys/class/hwmon/hwmon0/fan1_input # Restart fan control service sudo systemctl restart fan-control.service # View service logs sudo journalctl -u fan-control.service -f
